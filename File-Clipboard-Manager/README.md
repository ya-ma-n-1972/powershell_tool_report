# File Clipboard Manager

プロジェクトフォルダのファイルをフィルタリングし、クリップボードに一括登録するPowerShellツールです。

`.gitignore` のルールに基づいてファイルを自動選別し、Claude AI等のチャットボットへコードを送信する作業を効率化します。

## 主な機能

### フォルダベースのファイル管理

- プロジェクトフォルダを指定すると、配下のファイルを再帰的に取得
- `.gitignore` のルールを適用してファイルを自動フィルタリング（ON/OFF切替可能）
- バイナリ・メディア・アーカイブ等の不要な拡張子を除外（ネガティブリスト方式）
- ロックファイル（`package-lock.json` 等）を除外
- フォルダ履歴を最大10件保持し、プロジェクト間の切替が容易
- 先頭が `.` のディレクトリ（`.git`, `.github`, `.vscode` 等）は常に除外

### クリップボード登録

本ツールには3つのクリップボード登録方式があります。

**ファイルパス方式** -- フィルタリングされたファイルをエクスプローラー互換のファイルドロップリストとしてクリップボードに登録します。エクスプローラーへの貼り付け（Ctrl+V）が可能です。

**フェンス付きテキスト方式** -- 各ファイルの内容をファイルパスとMarkdownコードフェンス（` ``` `）で囲んだテキストとしてクリップボードに登録します。Claude AIのProject KnowledgeやチャットへのコードSnippet送信に最適です。

**クリップボードへのフェンス追加** -- 既にクリップボードにコピー済みのテキストやファイル（1件）に対して、コードフェンスを追加して再登録します。ちょっとしたコードスニペットの整形に便利です。

### その他の機能

- **ファイル詳細ウィンドウ**: フィルタリング結果の一覧表示、選択ファイルのみのクリップボード登録・フェンス付き登録
- **BOM付きUTF-8変換**: 選択したテキストファイルをBOM付きUTF-8に変換（ファイル直接上書き）
- **コード更新**: プロジェクト内のファイル変更後、ボタン一つでフィルタリングを再実行
- **タスクトレイ常駐**: 起動中はタスクトレイにアイコンを表示

## 動作環境

- Windows 10/11
- PowerShell 5.1 以降
- Python 3.x
- pathspec ライブラリ（`pip install pathspec`）

## インストール

1. リポジトリをクローンまたはダウンロード
2. `File-Clipboard-Manager.ps1`、`filter.py` を同じフォルダに配置
3. pathspec をインストール

```
pip install pathspec
```

初回起動時に `settings.json` が自動生成されます。

## 使い方

### 起動

PowerShellから以下のコマンドを実行：

```powershell
.\File-Clipboard-Manager.ps1
```

または、エクスプローラーから右クリック → 「PowerShellで実行」

起動時にPythonとpathspecの存在を自動チェックします。不足している場合はエラーメッセージが表示されます。

### 基本操作

#### 1. プロジェクトフォルダの選択

1. 「フォルダ選択」ボタンをクリック
2. プロジェクトフォルダを選択
3. `.gitignore` と除外ルールに基づいてファイルが自動フィルタリングされる
4. メッセージエリアに対象ファイル数が表示される

一度選択したフォルダは履歴に保存され、ComboBoxから切替可能です（最大10件）。

#### 2. クリップボードへ登録（ファイルパス方式）

1. 「クリップボードへ」ボタンをクリック
2. エクスプローラーなどに貼り付け（Ctrl+V）

#### 3. フェンス付きでクリップボードへ登録

1. 「フェンス付きクリップボード」ボタンをクリック
2. テキストエディタやチャットに貼り付け（Ctrl+V）

出力形式の例：

```
ファイルパス: D:\project\src\main.py
` `` 
def main():
    print("hello")
` ``

ファイルパス: D:\project\src\utils.py
` ``
def helper():
    pass
` ``
```

（実際の出力ではスペースは入りません）

#### 4. クリップボード内容にフェンス追加

1. テキストまたはファイルをコピー（Ctrl+C）
2. 「クリップボードにフェンス追加」ボタンをクリック
3. フェンスで囲まれた状態でクリップボードに再登録される

対応形式：クリップボードにコピーされたテキスト、またはエクスプローラーでコピーしたテキストファイル（1ファイルのみ、最大30MB）

#### 5. ファイル詳細の確認と個別操作

1. 「ファイル詳細」ボタンをクリック
2. フィルタリング結果のファイル一覧が表示される
3. ファイルを選択して以下の操作が可能：
   - 「クリップボードへ」-- 選択ファイルのみをファイルパス方式で登録
   - 「フェンス付き」-- 選択ファイルのみをフェンス付きで登録
   - 「BOM変換」-- BOM付きUTF-8に変換

#### 6. .gitignore フィルタリングの切替

メインウィンドウの「.gitignore を有効にする」チェックボックスでON/OFFを切替できます。

- ON（デフォルト）: `.gitignore` のルールを適用。`.gitignore` が存在しないフォルダではエラーになります。
- OFF: `.gitignore` を無視し、拡張子・ファイル名の除外ルールのみ適用。

なお、先頭が `.` のディレクトリ（`.git`, `.github`, `.vscode` 等）はこの設定に関わらず常に除外されます。

#### 7. コード更新

プロジェクト内でファイルの追加・削除・`.gitignore` の変更があった場合、「コード更新」ボタンでフィルタリングを再実行できます。

## 設定ファイル（settings.json）

初回起動時にデフォルト値で自動生成されます。手動編集も可能です。

```json
{
  "gitignore_enabled": true,
  "folder_history": [],
  "exclude_extensions": [
    ".png", ".jpg", ".jpeg", ".gif", ".bmp", ".svg", ".ico", ".webp",
    ".tiff", ".tif", ".psd", ".ai", ".eps", ".raw", ".heic", ".avif", ".icns",
    ".mp4", ".mp3", ".wav", ".flac", ".aac", ".ogg", ".avi", ".mov",
    ".mkv", ".wmv", ".m4a", ".m4v", ".webm", ".wma", ".m4p", ".3gp",
    ".exe", ".dll", ".so", ".dylib", ".bin", ".obj", ".class",
    ".apk", ".aab", ".ipa", ".jar", ".war", ".ear", ".jmod",
    ".zip", ".tar", ".gz", ".7z", ".rar", ".bz2", ".xz",
    ".pdf", ".doc", ".docx", ".xls", ".xlsx", ".ppt", ".pptx",
    ".db", ".sqlite", ".sqlite3",
    ".o", ".a", ".lib", ".pyc", ".pyo"
  ],
  "exclude_filenames": [
    "Cargo.lock", "package-lock.json", "yarn.lock", "pnpm-lock.yaml",
    "poetry.lock", "Pipfile.lock", "Gemfile.lock", "composer.lock",
    "go.sum", "packages.lock.json", "Package.resolved", "pubspec.lock"
  ]
}
```

各項目の説明：

- **gitignore_enabled**: `.gitignore` フィルタリングの有効/無効
- **folder_history**: 過去に選択したフォルダのパス（最大10件、自動管理）
- **exclude_extensions**: 除外する拡張子のリスト（画像・メディア・バイナリ・アーカイブ等）
- **exclude_filenames**: 除外するファイル名のリスト（ロックファイル等）

除外リストは用途に応じて手動で追加・削除できます。

## フィルタリングの仕組み

ファイルのフィルタリングは以下の順序で適用されます。

1. プロジェクトフォルダ配下のファイルを再帰的に取得（先頭が `.` のディレクトリは常に除外）
2. `.gitignore` が有効な場合、pathspec ライブラリでルールを適用
3. `exclude_extensions` に該当する拡張子のファイルを除外
4. `exclude_filenames` に該当するファイル名のファイルを除外

filter.py は単体でテストモードとしても実行できます：

```
python filter.py
```

引数なしで実行すると、pathspecの動作確認と任意の `.gitignore` でのフィルタリングテストが対話形式で行えます。

## トラブルシューティング

### スクリプトが実行できない

PowerShellの実行ポリシーを確認してください：

```powershell
Get-ExecutionPolicy
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 「Pythonが見つかりません」と表示される

PythonがPATHに通っているか確認してください。コマンドプロンプトで `python --version` が実行できれば問題ありません。

### 「pathspecが見つかりません」と表示される

以下のコマンドでインストールしてください：

```
pip install pathspec
```

### 「.gitignoreが見つかりません」と表示される

対象フォルダのルートに `.gitignore` が存在しない場合に表示されます。`.gitignore` を作成するか、「.gitignore を有効にする」チェックボックスをOFFにしてください。

### フェンス付きクリップボードで一部のファイルが含まれない

UTF-8として読み込めないファイルはスキップされます。スキップが発生した場合はメッセージに「○件スキップ」と表示されます。

## 注意事項

### BOM付きUTF-8変換について

「BOM変換」機能はファイルを直接上書きします。バックアップは作成されませんので、重要なファイルを変換する際は事前に手動でバックアップを取ってください。

### セキュリティについて

本ツールはネットワーク通信を行わず、ローカルファイルシステムのみを操作します。ただし、settings.json のパス情報に基づいてファイルを読み込む設計のため、信頼できない第三者が作成した settings.json を使用しないでください。

### 想定環境

本ツールは個人のローカル環境での使用を想定しています。ファイルサイズの上限チェックは実装していないため、巨大なファイルを含むフォルダを対象にするとメモリ不足になる可能性があります。主にソースコードや設定ファイルなど、テキストファイル中心のプロジェクトでの使用を推奨します。
